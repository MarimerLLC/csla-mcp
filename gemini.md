# Gemini CLI Instructions

Welcome to the `csla-mcp` repository! To ensure Gemini provides relevant and high-quality suggestions, please follow these guidelines:

## Coding Style
- Use consistent indentation (4 spaces).
- Prefer explicit variable names.
- Follow C# conventions for naming and structure.

## Documentation
- Add XML comments to public classes and methods.
- Use summary tags for method descriptions.

## Best Practices
- Write clean, maintainable code.
- Avoid hardcoding values; use configuration where possible.
- Implement error handling and logging.

## CSLA Specific Tooling
This project has custom tools to interact with CSLA documentation and samples.
- Use the `search` tool to find relevant CSLA patterns.
- Use the `fetch` tool to retrieve specific CSLA code examples.

## Pull Requests
- Ensure code is tested before submitting.
- Include a clear description of changes.

## Gemini Usage
- Use Gemini to generate boilerplate, tests, and documentation.
- Review and edit Gemini suggestions for accuracy and security.

## Change logs and change documents
- Create all change logs and change documents in the `changelogs` folder.
- When documenting fixes or other changes, put the documents in the `changelogs` folder.

Thank you for contributing!
